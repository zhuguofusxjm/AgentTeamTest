# 币安U本位合约量化分析系统 — 操作指导手册

## 一、环境准备

### 1.1 安装依赖

```bash
pip install flask requests
```

Python 版本要求：3.11+

### 1.2 配置环境变量

```bash
# DeepSeek API Key（用于AI二次分析）
export DEEPSEEK_API_KEY="sk-xxxxx"
```

### 1.3 配置 Server酱（微信推送）

1. 访问 https://sct.ftqq.com 用微信扫码登录
2. 获取 SendKey
3. 在 `monitor.py` 中填入 SENDKEY（已配置则跳过）

## 二、首次部署

### 2.1 全量数据采集

```bash
# 第1步：采集费率 + K线数据（近2年，耗时较长）
python fetch_funding_data.py

# 第2步：采集控仓数据（近30天）
python fetch_position_data.py
```

首次运行会自动创建 `funding_rate.db` 数据库。费率+K线采集可能需要数小时（取决于网络）。

### 2.2 验证数据

```bash
# 检查数据量
python -c "from db import get_conn; c=get_conn(); print('费率:', c.execute('SELECT COUNT(*) FROM funding_rate_history').fetchone()[0]); print('K线:', c.execute('SELECT COUNT(*) FROM kline_1h').fetchone()[0]); print('控仓:', c.execute('SELECT COUNT(*) FROM position_data').fetchone()[0])"
```

### 2.3 运行回测（可选）

```bash
# 多进程并行回测，约100秒
python backtest.py
```

## 三、日常使用

### 3.1 一键启动（推荐）

```bash
python start.py
```

同时启动 Web 服务和信号监控，按 Ctrl+C 一键停止全部。

启动后：
- Web页面：http://localhost:5000
- 信号扫描：每30分钟自动执行
- 持仓跟踪：每15分钟自动执行
- 数据更新：每次扫描前自动增量更新

### 3.2 分开启动

如果需要单独控制，也可以分开启动：

```bash
# 终端1：Web 服务
python app.py

# 终端2：信号监控
python monitor.py

# 自定义参数
python monitor.py --interval 3600      # 信号扫描改为1小时
python monitor.py --min-score 50       # 只推送50分以上的强信号
python monitor.py --once               # 只扫描一次，不循环
```

### 3.3 Web 页面功能

访问 `http://localhost:5000`

- 主表格：所有币种的费率、信号、年化等
- 信号列：点击查看触发原因和数据支撑
- 查看历史：4区域联动图表（K线/成交量/费率/控仓）
- 跟踪按钮：添加持仓跟踪
- 右上角"持仓跟踪"：打开右侧跟踪侧边栏
- 右上角"回测报告"：查看策略回测结果

### 3.4 添加持仓跟踪

**方式1：Web页面**
1. 在主页找到目标币种
2. 点击"跟踪"按钮
3. 填入方向（多/空）、入场价、止损价、止盈价
4. 点击"确认跟踪"

**方式2：命令行**
```bash
python -c "from db import init_db, add_tracked_position; init_db(); add_tracked_position('BTCUSDT', '多', 94000, 91000, 97000, '测试')"
```

### 3.5 查看跟踪状态

- Web页面：点击右上角"持仓跟踪"按钮
- 命令行：`python -c "from db import get_active_tracks; print(get_active_tracks())"`
- 微信：monitor.py 每15分钟自动推送跟踪报告

### 3.6 关闭跟踪

- Web页面：在跟踪侧边栏点击"关闭"按钮
- 自动关闭：价格触发止损/止盈时自动标记关闭

## 四、数据更新

### 4.1 自动更新（推荐）

monitor.py 运行时会自动增量更新所有数据，无需手动操作。

### 4.2 手动更新

```bash
# 增量更新费率+K线
python fetch_funding_data.py

# 增量更新控仓数据
python fetch_position_data.py

# 单独更新K线
python fetch_kline_data.py

```

## 五、信号解读

### 5.1 信号分类

| 类型 | 颜色 | 含义 |
|------|------|------|
| 确认信号 80+ | 红色 | 强信号，指标已极端 |
| 确认信号 50-79 | 黄色 | 中信号 |
| 早期信号 | 蓝色"早XX" | 蓄力阶段，适合提前关注 |
| ↑ 绿色箭头 | 绿色 | 偏多方向 |
| ↓ 红色箭头 | 红色 | 偏空方向 |

### 5.2 各信号含义

| 信号 | 含义 | 实战意义 |
|------|------|----------|
| 成交量异常 | 成交量暴增+主买/主卖占比极端 | 有大资金在行动 |
| 大户持仓极端 | 大户仓位严重偏向一侧 | 大户在押注方向 |
| 持仓量急增 | 大量资金进场但价格没动 | 多空僵持，即将选择方向 |
| 大户散户分歧 | 大户和散户方向相反 | 散户往往被收割 |
| 费率极端 | 多头/空头过度拥挤 | 容易引发踩踏反转 |
| 波动率压缩 | 价格波动极度缩小 | 弹簧蓄力，突破后大幅运动 |
| 持仓量缓增 | 资金连续多天悄悄进场 | 有人在布局 |
| 大户悄悄建仓 | 大户持仓比缓慢偏移 | 大户在悄悄加仓 |
| 成交量萎缩 | 成交量骤降+价格窄幅 | 暴风雨前的平静 |
| 价格收敛 | 三角形态收敛 | 突破在即 |
| 布林带收口 | 价格被压缩在极窄区间 | 即将选择方向 |
| 费率趋势变化 | 费率逐步走高/走低 | 情绪在酝酿 |

### 5.3 历史图表中的控仓数据

第4区域的4条线：
- **蓝色面积图**：持仓量（全市场未平仓合约总价值USDT）
- **橙色实线**：大户持仓多空比（>1 大户偏多，<1 偏空）
- **紫色实线**：大户账户多空比（做多/做空的大户人数比）
- **绿色虚线**：全市场多空比（所有散户的多空比）

大户线和全市场线方向相反时，是比较强的信号。

## 六、微信推送消息说明

### 6.1 信号推送

收到两类消息：
- **确认信号 N 个**：指标已极端的币种，按大盘>中盘>小盘排序
- **早期信号(关注) N 个**：蓄力阶段的币种

每个币种包含：
- 分组标签（大盘/中盘/小盘）
- 当前价格和24h涨跌
- 触发的每个信号名称、分数、原因、数据支撑
- 建议止损/止盈价位
- DeepSeek 独立分析（是否同意方向、风险点、可信度）

### 6.2 持仓跟踪推送

每15分钟推送一次，包含：
- 当前价格和浮动盈亏
- 距止损/止盈的距离
- 相比上次的增量变化（价格、盈亏、费率、多空比）
- DeepSeek 跟进建议（好转/恶化、持有/加仓/减仓/平仓）
- 触发止损/止盈时自动提醒

## 七、审计与复盘

### 7.1 审计日志文件

所有 LLM 交互日志存储在 `tracks/` 目录下，分两类：

**全市场扫描日志**（每次扫描生成）：
```
tracks/scan_confirmed_2026-05-04_10-30-00.json   # 确认信号扫描
tracks/scan_early_2026-05-04_10-30-00.json       # 早期信号扫描
```

包含：扫描时间、所有推送币种的信号详情和价格、DeepSeek 完整 prompt/response/token 用量。

**持仓跟踪日志**（每个跟踪任务一个文件，持续追加）：
```
tracks/BTCUSDT_1.json
tracks/LUMIAUSDT_2.json
```

包含：入场信息、每轮市场快照、增量变化、K线数据、DeepSeek 完整 prompt/response/token 用量、关闭信息。

### 7.2 复盘方法

将跟踪 JSON 文件直接发给 LLM 进行复盘分析：

```
请复盘以下交易记录，分析：
1. 入场时机是否合理
2. 持仓过程中哪些信号被忽略了
3. 出场时机是否最优
4. AI建议的准确率如何
5. 改进建议

[粘贴 JSON 内容]
```

### 7.3 数据库查询

```bash
# 查看所有跟踪记录
python -c "from db import get_all_tracks; import json; print(json.dumps(get_all_tracks(), ensure_ascii=False, indent=2))"

# 查看某个跟踪的分析历史
python -c "from db import get_track_history; import json; print(json.dumps(get_track_history(1), ensure_ascii=False, indent=2))"
```

### 7.4 审计日志查看

```bash
# 查看最近的扫描日志
ls -lt tracks/scan_*.json | head -5

# 查看某个跟踪的完整日志
cat tracks/BTCUSDT_1.json | python -m json.tool
```

## 八、故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 采集数据失败 | 网络无法访问币安 | 检查代理设置 |
| 推送失败 | SendKey 过期或错误 | 重新获取 SendKey |
| DeepSeek 分析失败 | API Key 无效或余额不足 | 检查 DEEPSEEK_API_KEY |
| 信号列为空 | 数据未采集或过旧 | 运行 fetch 脚本更新数据 |
| 跟踪不推送 | monitor.py 未运行 | 启动 monitor.py |
| 页面报错 entry_signals | 旧数据库缺少新列 | 重启 app.py（自动加列） |
| 回测太慢 | 数据量大 | 已支持多进程，默认8核 |

## 九、注意事项

1. **网络要求**：需要能访问 `fapi.binance.com` 和 `api.deepseek.com`
2. **控仓数据限制**：币安API仅支持最近30天，每次采集会累积存储
3. **信号非投资建议**：回测胜率约43.6%，需结合自身判断
4. **资金安全**：当前系统不涉及自动交易，不需要币安交易权限API Key
5. **推送频率**：Server酱免费版每天限制推送次数，注意控制
6. **磁盘空间**：全量数据约数百MB到数GB，定期检查
7. **时区**：所有时间为UTC+8（北京时间）
