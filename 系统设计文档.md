## 二、文件清单

| 文件 | 职责 |
|------|------|
| `funding_rate.py` | 币安API封装（费率/K线/持仓量/多空比，共10个接口+10个分页版本） |
| `db.py` | SQLite数据库操作（6张表，25+个函数） |
| `fetch_funding_data.py` | 全量采集脚本（费率+K线，近2年） |
| `fetch_kline_data.py` | 独立K线采集脚本 |
| `fetch_position_data.py` | 控仓数据采集脚本（4个指标，近30天） |
| `coin_groups.py` | 币种分组（按成交量分大盘/中盘/小盘，每组独立参数） |
| `signals.py` | 信号识别引擎v3（6个确认信号+6个早期信号，参数化，含数据支撑） |
| `backtest.py` | 策略回测引擎（多进程并行，分组统计） |
| `monitor.py` | 实时监控（数据更新+信号扫描+持仓跟踪+DeepSeek分析+微信推送+审计日志） |
| `app.py` | Flask Web服务（15个路由，含持仓跟踪管理） |
| `start.py` | 一键启动脚本（同时启动app.py和monitor.py） |
| `templates/index.html` | 主页（费率表格+信号列+历史图表+右侧跟踪侧边栏） |
| `templates/backtest.html` | 回测报告页（统计卡片+图表+交易记录） |
| `tracks/` | 审计日志目录（扫描日志+跟踪日志，JSON格式） |

## 三、数据库设计

### 3.1 表结构

#### funding_info — 币种元数据
```
symbol TEXT PK | funding_rate_cap TEXT | funding_rate_floor TEXT | funding_interval_hours INT
```

#### funding_rate_history — 历史资金费率
```
symbol TEXT PK | funding_rate TEXT | funding_time INT PK | funding_time_str TEXT
索引: idx_fh_symbol, idx_fh_time
```

#### kline_1h — 1小时K线
```
symbol TEXT PK | open_time INT PK | open/high/low/close TEXT | volume TEXT
close_time INT | open_time_str TEXT | quote_volume TEXT | trades INT
taker_buy_volume TEXT | taker_buy_quote_volume TEXT
索引: idx_kline_symbol
```

#### position_data — 控仓数据
```
symbol TEXT PK | timestamp INT PK | metric_type TEXT PK
timestamp_str TEXT | value1 TEXT | value2 TEXT | value3 TEXT
metric_type: oi / top_position / top_account / global_account
索引: idx_pos_symbol, idx_pos_time
```

#### tracked_positions — 持仓跟踪
```
id INT PK AUTO | symbol TEXT | direction TEXT | entry_price REAL
stop_loss REAL | take_profit REAL | status TEXT | created_at TEXT
closed_at TEXT | close_reason TEXT | notes TEXT | entry_signals TEXT
```

#### track_history — 跟踪分析历史
```
id INT PK AUTO | track_id INT FK | timestamp TEXT | price REAL | pnl REAL
funding_rate REAL | top_ratio REAL | global_ratio REAL
oi_value REAL | volume_24h REAL | snapshot TEXT | ai_analysis TEXT
```

### 3.2 数据来源

| 数据 | 币安API | 历史深度 | 更新频率 |
|------|---------|----------|----------|
| 费率元数据 | /fapi/v1/fundingInfo | 实时 | 每次采集 |
| 历史费率 | /fapi/v1/fundingRate | 2年 | 每4h |
| 1h K线 | /fapi/v1/klines | 2年 | 每1h |
| 持仓量 | /futures/data/openInterestHist | 30天 | 每1h |
| 大户持仓比 | /futures/data/topLongShortPositionRatio | 30天 | 每1h |
| 大户账户比 | /futures/data/topLongShortAccountRatio | 30天 | 每1h |
| 全市场多空比 | /futures/data/globalLongShortAccountRatio | 30天 | 每1h |

## 四、信号系统设计

### 4.1 币种分组

按最近24h成交量自动分组，每组使用不同参数：

| 分组 | 占比 | 成交量阈值 | 止损/止盈 | 最低分数 |
|------|------|-----------|-----------|----------|
| 大盘 | top 10% | 成交量最高 | -2%/+3% | 35 |
| 中盘 | 10%-40% | 中等 | -3%/+5% | 30 |
| 小盘 | 后60% | 较低 | -4%/+7% | 30 |

### 4.2 确认型信号（6个）

| 信号 | 条件（中盘参数） | 分值 | 含义 |
|------|-----------------|------|------|
| 成交量异常 | 量>3x均值 且 主买>70%或<30% | 25 | 大资金在行动 |
| 大户持仓极端 | 大户比>2.5 或 <0.4 | 25 | 大户押注方向 |
| 持仓量急增 | 24h OI涨>20% 且 价格变化<2% | 20 | 多空僵持蓄力 |
| 大户散户分歧 | 大户和散户方向明显相反 | 18 | 散户可能被收割 |
| 费率极端 | 费率绝对值>0.15% | 10 | 拥挤踩踏风险 |
| 波动率压缩 | 近12h ATR<7天ATR的35% | 10 | 弹簧蓄力 |

### 4.3 早期信号（6个）

| 信号 | 条件 | 分值 | 含义 |
|------|------|------|------|
| 持仓量缓增 | 连续3天OI递增>8%，价格平稳 | 18 | 资金悄悄进场 |
| 大户悄悄建仓 | 大户比缓慢偏移>15%，未到极端 | 18 | 大户在布局 |
| 成交量萎缩 | 量降至前期35%以下，价格窄幅 | 15 | 暴风雨前的平静 |
| 价格收敛 | 低点抬高或高点降低，三角收敛 | 15 | 突破在即 |
| 布林带收口 | 带宽收窄至前期25%以下 | 10 | 价格被压缩 |
| 费率趋势变化 | 费率逐步走高/走低但未过热 | 10 | 情绪在酝酿 |

### 4.4 信号过滤规则

- 布林带收口和波动率压缩单独触发不算，需要有其他信号共振
- 方向一致性加分：多个信号方向一致时额外+10分
- 每个信号带具体数据支撑和原因说明

### 4.5 回测结果（601个币种，2年数据）

| 指标 | 数值 |
|------|------|
| 总交易数 | 1,540 |
| 胜率 | 43.6% |
| 平均收益 | +0.10% |
| 盈亏比 | 1.38 |
| 大盘胜率/收益 | 43.8% / +0.18% |
| 中盘胜率/收益 | 39.8% / +0.04% |
| 小盘胜率/收益 | 44.4% / +0.11% |

## 五、监控与推送设计

### 5.1 信号扫描（每30分钟）

```
10线程并发更新数据 → 计算所有币种信号 → 分类(确认/早期)
→ 按分组排序(大盘>中盘>小盘) → DeepSeek二次分析(10个一批)
→ 保存扫描审计日志 → 微信推送
```

推送规则：
- 确认信号和早期信号分开推送（两条消息）
- 每次最多推送10个
- 同一币种4小时内不重复推送
- 最低推送分数30

DeepSeek交互方式：筛出的10个币种打包成一个JSON，一次性发给DeepSeek分析，返回对每个币种的逐一判断。

### 5.2 持仓跟踪（每15分钟）

```
获取活跃跟踪列表 → 获取市场数据 → 计算增量变化
→ 读取历史分析记录(最近5次) → 打包完整上下文给DeepSeek(逐个)
→ 保存快照到DB+JSON文件 → 推送微信
```

DeepSeek每次收到的累积上下文：
1. 入场信息（触发的信号、价格、方向、时间）
2. 当前状态（价格、盈亏、费率、多空比、持仓量）
3. 增量变化（vs上次分析：价格变化、盈亏变化、费率变化、多空比变化）
4. 最近12h K线原始数据
5. 历史分析记录（最近5次的价格、盈亏和AI建议摘要）

### 5.4 启动方式

使用 `start.py` 一键启动所有服务：

```bash
python start.py
```

同时启动：
- `app.py` — Web服务 (http://localhost:5000)
- `monitor.py` — 信号扫描(30min) + 持仓跟踪(15min)

按 Ctrl+C 一键停止全部。任一进程异常退出时自动停止另一个。

## 六、前端设计

### 6.1 主页 (index.html)

- 费率总览表格：币种、信号、费率、年化、结算周期
- 信号列：确认信号(红/黄) + 早期信号(蓝) + 方向箭头
- 点击信号弹出详情面板（每个条件的原因和数据支撑）
- 历史图表弹窗（4区域联动）：K线 / 成交量+主买量 / 费率 / 持仓量+多空比
- 右侧持仓跟踪侧边栏
- 搜索过滤、列排序、全屏模式

### 6.2 回测报告 (backtest.html)

- 汇总卡片（胜率、盈亏比、平均收益等）
- 各信号胜率对比图
- 确认信号 vs 早期信号对比图
- 详细交易记录表
